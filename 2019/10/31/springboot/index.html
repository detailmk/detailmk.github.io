springboot

定义

  • 它使用 “习惯优于配置” （项目中存在大量的配置，此外还内置一个习惯性的配置，让你无须）的理念让你的项目快速运行起来。
  • 它并不是什么新的框架，而是默认配置了很多框架的使用方式，就像 Maven 整合了所有的 jar 包一样，Spring Boot 整合了所有框架

核心特性

  • 极低学习成本
  • 可独立运行spring项目
  • “习惯优于配置”,极大提高开发效率
  • 极其简单的组件依赖,自动发现和自动装配
  • 提供运行时的应用监控
  • 与分布式架构和云计算的天然集成

启动流程

  1. 加载配置文件
  2. 自动装配
  3. 加载组件
  4. 应用初始化

Spring Boot是什么？

底层由Spring框架进行构建，然后通过Spring Boot 进行牵引，其优点：

1、 简化XML配置

2、 快速构建Spring项目

3、 通过‘starter’能够快速集成其他技术框架

4、 内嵌Tomcat、Jetty等web 服务器，从而build项目时不需要单独添加Tomcat等服务器

5、 提供生产级别的监控指标数据

Spring 优缺点?

Spring优点

1、 免费开源框架

2、 拥有IOC/DI和AOP等组件能够简化开发者开发，同时提高系统性能，降低对象与对象之间的耦合

3、 能够方便集成其他第三方框架或中间件技术，所以称之为“粘合剂”

4、 目前Spring提供非常多的第三方组件比如：MVC框架、Spring Data、Spring AMQP等

Spring缺点

1、 重量级框架，包含很多的组件，其他轻量级IOC框架：谷歌Guice

2、 繁琐的xml配置

3、 集成其他组件比较复杂

4、 必须依赖web容器启动IOC容器

Spring Boot启动流程

掌握@SpringBootApplication启动类注解

==@SpringBootApplication== = ==@SpringBootConfiguration== + ==@ComponentScan== + ==@EnableAutoConfiguration==，其中：
==@SpringBootConfiguration==：标记此类是一个配置类，可以结合@Bean完成一些创建bean的操作
==@ComponentScan==是一个扫描注解，默认情况下扫描当前目录以及子孙目录，如果需要指定扫描的路径可以设置value值，比如==@ComponentScan==(“com.mk”)，一般就用默认（约定由于配置）
==@EnableAutoConfiguration== 自动装载配置注解

初始化Spring的上下文ApplicationContext

通过执行SpringApplication.run()就会创建ConfigurableApplicationContext对象

启动Tomcat服务器

第一步：通过自动装载（EnableAutoConfiguration）获取到Tomcat相关的工厂bean和配置bean

第二步：在初始化容器之后的onRefresh()方法中在加载Tomcat相关的bean对象（工厂bean）

第三步：通过工厂bean获取一个WebServer对象，这个WebServer就包含有Tomcat，然后启动Tomcat

MVC和三层架构

MVC(Model、View、Controller)：是一种解耦思想，就是在做web开发时将页面和业务逻辑进行分离，交互时将数据进行封装，能够让web开发流程变得更加清晰，各层职责更加分明。

三层架构：web层（控制层）、service层（业务逻辑处理层）、dao（数据访问层），在web开发中针对MVC中的C进行分离和封装, SpringMVC解决了Servlet处理请求，给出响应的问题, MyBatis解决了数据库访问的问题

视图（view）层：该层是直接面向用户，它能够给用户提供操作界面，可以是程序的外壳。
数据（Model）层：该层是程序底层，负责封装程序需要的数据。
控制（Controller）层：该层是中间层，主要是接收视图层的操作指令，同时获取数据层的数据， 并根据一定的计算逻辑将结果封装到数据层并将结果返回给视图层，视图层在展示给用户。

SpringMVC请求处理流程

1、首先用户发送请求,请求被SpringMvc前端控制器（DispatherServlet）捕获；

2、前端控制器(DispatherServlet)对请求URL解析获取请求URI,根据URI, 调用HandlerMapping；

3、前端控制器(DispatherServlet)获得返回的HandlerExecutionChain（包括Handler对象以及Handler对象对应的拦截器）；

4、DispatcherServlet 根据获得的HandlerExecutionChain，选择一个合适的HandlerAdapter。（附注：如果成功获得HandlerAdapter后，此时将开始执行拦截器的preHandler(…)方法；

5、HandlerAdapter根据请求的Handler适配并执行对应的Handler；

HandlerAdapter(提取Request中的模型数据，填充Handler入参，开始执行Handler（Controller)。 在填充Handler的入参过程中，根据配置，Spring将做一些额外的工作：

· HttpMessageConveter： 将请求消息(如Json、xml等数据)转换成一个对象，将对象转换为指定的响应信息。比如 json，默认使用MappingJackson2HttpMessageConverter转换

· 数据转换：对请求消息进行数据转换。如String转换成Integer、Double等

· 数据格式化：对请求消息进行数据格式化。 如将字符串转换成格式化数字或格式化日期等

· 数据验证： 验证数据的有效性(长度、格式等)，验证结果存储到BindingResult或Error中）

6、Handler执行完毕，返回一个ModelAndView(即模型和视图)给HandlerAdaptor

7、HandlerAdaptor适配器将执行结果ModelAndView返回给前端控制器。

8、前端控制器接收到ModelAndView后，请求对应的视图解析器。

9、视图解析器解析ModelAndView后返回对应View;

10、渲染视图并返回渲染后的视图给前端控制器

11、最终前端控制器将渲染后的页面响应给用户或客户端。
